# SERIE 集客改善 作業指示書

作成日: 2026-07-22
対象施設: Lodgers Bldg SERIE（栃木県那須塩原市中央町3-12 / JR黒磯駅近く）
発行: 芹江匡晋（社長）

---

## 0. この指示書の使い方

- **VS Code の AI（Claude Code）に読ませる場合**: このファイル全体をコンテキストに入れ、「タスクA から順に実行して」と指示する。
- **人間（芹江・小筆）が作業する場合**: 「👤人間」と書かれた項目を上から実行する。「🤖AI」項目はAIに任せる。
- 各タスクに **完了条件（DoD）** があるので、そこを満たしたらチェックを入れる。

---

## 1. 背景と目的（なぜやるか）

| 項目 | 内容 |
|---|---|
| 問題 | 客室稼働率が去年（90%以上）から今年 **半分以下** に急落 |
| 去年との違い | 去年は Booking.com・Airbnb の2サイトのみ。今年 VRBO・Trip.com を追加 |
| 判明した事実 | 「那須塩原 ホテル」「那須 格安」等の**一般検索でSERIEが一切ヒットしない**。Booking.comでは「黒磯」ピンポイント指定でしか出てこず、「那須塩原市」エリア検索の対象から外れている疑い |
| 目的 | **固有名詞（SERIE）ではなく「那須塩原」「那須」の一般検索・エリア検索でヒットさせ、稼働率を回復する** |

### 集客の入口は3つしかない（ここが主戦場）

| 優先 | 入口 | 現状 |
|---|---|---|
| ★★★ | Googleビジネスプロフィール（Googleマップ枠） | 未登録の疑い |
| ★★★ | OTA内「那須塩原」エリア検索（Booking/Airbnb等） | エリア紐付けが壊れている疑い |
| ★★☆ | OTA経由のGoogle自然検索 | ②が直れば自動改善 |

> 個人宿がGoogle自然検索で単独1ページ目に出るのは不可能（楽天・じゃらん・Booking等が独占）。戦うのは①②。

## 2. タスク一覧

### タスクA｜現状診断（🤖AI が自動実行）

**目的**: 修正前のベースラインを記録し、どこで露出が切れているか特定する。

- [ ] A-1. 各サイトで以下のキーワード検索を行い、SERIEが表示されるか・何番目かを記録
  - Google: `那須塩原 ホテル 格安` / `那須 ホテル 格安` / `黒磯 ホテル`
  - Booking.com: 目的地「那須塩原市」/ 目的地「黒磯」
  - Airbnb・VRBO・Trip.com: エリア「那須塩原」
- [ ] A-2. 競合（黒磯・那須塩原駅5km圏の民泊・ゲストハウス）の施設名・料金・レビュー数・星評価を5件収集
- [ ] A-3. 結果を `output/shukyaku_shindan_YYYYMMDD.md` に表形式で保存

**完了条件**: 各キーワードでの掲載順位（または「圏外」）が一覧化され、ファイル保存されている。

> 注意: Booking.com・Google Travel は Bot対策で自動取得が失敗することがある。取得できない場合は「👤人間が手動で検索し、スクリーンショットを提供」に切り替える。

---

### タスクB｜Booking.com のエリア・住所設定修正（👤人間 + 🤖AI）

**これが最優先。稼働率半減の主原因と見ている。**

- [ ] B-1.（👤人間）Booking.com エクストラネットにログイン → 「設備・情報」→「所在地・地図」を開く
- [ ] B-2.（👤人間）以下を確認し、スクリーンショットをAIに共有
  - 市区町村フィールドが **「那須塩原市」** になっているか（「黒磯」等の別表記になっていないか）
  - 郵便番号・住所が **中央町3-12** で正しいか
  - 地図ピンが実際の建物位置に置かれているか
- [ ] B-3.（👤人間）ズレていれば「那須塩原市」に修正・地図ピンを正しい位置へ
- [ ] B-4.（🤖AI）物件名・説明文の**地名キーワード**を強化する文案を作成（下記ルール参照）
  - 説明文の冒頭に「那須塩原」「那須」「黒磯」「JR黒磯駅」を自然に含める
  - タイトルは長すぎるキーワード羅列を避け、地名＋施設タイプを明快に
- [ ] B-5.（👤人間）AI作成の文案をエクストラネットに反映
- [ ] B-6.（👤人間）施設タイプ・星クラス欄が空欄でないか確認（空欄だと class ソートで圏外）

**完了条件**: Booking.comで目的地「那須塩原市」を選んだ検索結果にSERIEが表示される（B-1〜B-6反映後、24〜72時間で反映されることがある）。

### タスクC｜Googleビジネスプロフィール登録（👤人間 + 🤖AI）

**「那須塩原 ホテル」のGoogleマップ枠を無料で取りに行く。**

- [ ] C-1.（👤人間）https://business.google.com にアクセスし、SERIEを新規登録（既存なら「オーナー確認」）
- [ ] C-2.（👤人間）カテゴリを「旅館」または「ホテル」に設定、住所（中央町3-12）・電話・営業時間を入力
- [ ] C-3.（🤖AI）ビジネス説明文（750字以内）を「那須塩原」「那須」「黒磯」キーワード入りで作成
- [ ] C-4.（👤人間）外観・客室・周辺の写真を最低10枚アップロード
- [ ] C-5.（👤人間）ハガキ等での **オーナー確認（本人確認）** を完了させる

**完了条件**: Googleマップで「那須塩原 ホテル」検索時にSERIEがマップ上に表示される。

---

### タスクD｜他OTA（Airbnb・VRBO・Trip.com）のエリア確認（👤人間 + 🤖AI）

- [ ] D-1.（👤人間）各サイトの管理画面で住所・エリア設定が「那須塩原市」で正しいか確認
- [ ] D-2.（🤖AI）各サイトのタイトル・説明文に地名キーワードを入れた文案を作成
- [ ] D-3.（👤人間）反映

**完了条件**: 3サイトすべてで「那須塩原」エリア検索にSERIEが表示される。

### タスクE｜ランキング要因の底上げ（継続）

OTA内・Googleマップの表示順は「エリア紐付け」に加え、以下で決まる。

- [ ] E-1.（👤人間）宿泊客へのレビュー依頼を徹底（去年より分散したレビューを回復）
- [ ] E-2.（🤖AI / `pricing_agent.py`）競合料金を調査し、稼働回復期の推奨価格を提案
- [ ] E-3.（👤人間）写真の刷新・返信速度の改善（メッセージは24時間以内返信）

## 3. 地名キーワード文案ルール（🤖AI 共通ルール）

タイトル・説明文を作るときは以下を守る:

1. **必ず含める地名**: 那須塩原 / 那須 / 黒磯 / JR黒磯駅
2. **キーワードの羅列を避ける**（例:「那須の玄関口 黒磯の和風宿 Wi-Fi強」のような不自然な名前はNG）
3. 施設タイプ（旅館・ゲストハウス・アパートメント等）を明記
4. 各OTAの文字数制限に合わせる
5. 日本語版と英語版の両方を用意（外国人客対応）
